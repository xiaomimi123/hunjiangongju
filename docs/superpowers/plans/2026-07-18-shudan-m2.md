# 书单号复刻 M2（画面像 + 节奏像）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** 让生成成片在**画面版式、画风、剪辑节奏**上对齐源书单号（《书名》+作者常驻头 · 满屏厚涂油画配图 · 中英双语字幕 · @水印 · 源节奏）。

**Architecture:** 承接 M1（feat/shudan-m1）。拆解侧加 vision 识别源画风；生成侧给每段落「书名/作者/英文字幕」并按源节奏定时；模板 `worker/templates/booklist/indexHtml.ts` 重做为书单号版式。

**Tech Stack:** TS、Prisma、BullMQ、百炼 DashScope（qwen-vl 多模态）、HyperFrames、ffmpeg、vitest。

## Global Constraints

- 百炼 DashScope：非 LLM 能力走原生 `multimodal-generation`；模型名小写连字符（错=404）；北京地域。**每个新适配器实现前先 WebFetch 官方文档核对确切 model id / 请求体 / 响应结构**。
- vision 输入图片须 DashScope 可达 URL → 复用 M1 的 `publicAssetUrl`；本地 localhost 不可达 → 本地走 mock，真实验收在服务器。
- 本地 mock 打通逻辑与流水线 → 推服务器真跑；禁止未在本地打通就推。
- Mock 分支必须保留，整条流水线本地可跑（mock）。
- 现有模板契约 `BodyData{ size, overlay:{title,subtitle,watermark}, images[], segments:[{seqNo,startMs,endMs,subtitle,imageIndex}] }`；扩展要**向后兼容**（旧任务无新字段仍能渲染）。
- 测试：纯逻辑/解析用 vitest（DB 测试须容器内 `docker compose exec -T web npm test`）。渲染类改动跑 `indexHtml.test.ts`。
- 文档随代码更新。

## Task 1: vision 能力 + 拆解画风识别

**Files:**
- Create: `packages/db/src/ai/vision.ts`
- Create: `worker/src/gen/extractStyle.ts`
- Modify: `worker/src/gen/extractFramework.ts`（接入，写入 framework）
- Modify: `packages/db/src/ai/index.ts`（导出）
- Test: `packages/db/src/ai/vision.test.ts`

**Interfaces:**
- `describeImageStyle(imageUrls: string[]): Promise<{ imageStylePrompt: string; visualStyleType: string }>`（qwen-vl 多模态；mock 返回 `{imageStylePrompt:'厚涂油画质感,情绪化,统一画风', visualStyleType:'oil_painting'}`）
- `parseVisionStyle(raw: any): { imageStylePrompt: string; visualStyleType: string }`（纯解析）
- `extractStyleFromVideo(sourceVideoId: string, videoAbs: string): Promise<{imageStylePrompt, visualStyleType}>`（抽 3~5 帧→publicAssetUrl→describeImageStyle）

- [ ] **Step 0（必做）**：WebFetch qwen-vl 多模态文档，核对 model id（如 `qwen-vl-max` / `qwen3-vl-plus`）、图片输入字段（`input.messages[].content[].image` 用 URL）、响应结构。记录到报告。
- [ ] **Step 1**：写失败测试 `parseVisionStyle`（给定 qwen-vl 文本响应含"厚涂油画/暗调"→提取 imageStylePrompt，映射 visualStyleType）。

- [ ] **Step 3**：实现 `vision.ts`（`isDashScope` 分支走原生多模态，prompt 要求用一句中文描述画风+媒介+情绪，附 visualStyleType 词表让其归类；mock 分支）；`extractStyle.ts`（ffmpeg 抽帧到 /data、`publicAssetUrl` 签名、调 describeImageStyle）。
- [ ] **Step 4**：`extractFramework.ts` 接入：拆解时调 `extractStyleFromVideo`，把结果写入 framework 的 `imageStylePrompt`/`visualStyleType`（mock 或失败时回退现默认，不阻断）。
