# v2.2 P0 基座 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为「爆款视频拆解生成」v2.2 新产品搭好基座：新数据模型、AI 能力适配层（4 能力 + mock）、模型配置后台页、新流水线通用状态机——全部**增量新增**，不破坏现有可运行的东西。

**Architecture:** 沿用现仓库（Next.js 14 App Router + Prisma/PostgreSQL + BullMQ/Redis 的 monorepo：`packages/db`/`web`/`worker`）。P0 只**追加**新 Prisma 模型与新代码，不删旧混剪表/页/代码（拆旧属"切换上线"任务，见 spec §12）。AI 调用统一走一个位于 `packages/db` 的适配层（web 与 worker 均可导入），每种能力有独立 client，未接通/未启用时走 mock。密钥用现有 AES-256-GCM（`crypto`）加密，密钥派生自 `JWT_SECRET`。

**Tech Stack:** TypeScript、Prisma 5、Node 20 全局 `fetch`、vitest、现有 `@/lib/api`(handler)、`@/lib/auth`(requireRole/HttpError)、`@mixcut/db`。

## Global Constraints

- **只增不删**：P0 不改动/不删除现有 `Material`/`Script`/`Task`/`tags` 相关模型、接口、页面。新代码与旧代码并存，`next build` 与 `vitest` 始终通过。
- **表名不与旧表冲突**：新流水线的状态日志/质检用**新表** `render_status_logs`/`render_qc_reports`（旧 `task_status_logs`/`qc_reports` 保持不动）。spec §4 假定旧表已删故复用同名；P0 增量阶段用新表名避免碰撞，切换上线时再统一。
- **密钥加密**：所有 AI `apiKey` 落库前用 `encrypt()` 加密（AES-256-GCM，派生自 `JWT_SECRET`）。接口返回**绝不**回传明文 key，只回 `hasKey: boolean`。
- **能力标识固定四种**：`llm | image | tts | asr`。
- **Mock 优先可跑**：任一能力 `enabled=false` 或环境变量 `AI_MOCK=1` 时，对应 client 返回**结构合法的假数据**（假文案 / 极小合法 PNG / 极短静音 WAV / 固定假转写）；每个 client 自带 mock，**不互相路由**（见记忆 forgecast-mock-llm-per-capability）。
- **迁移方式**：非交互容器内用 `prisma migrate deploy` + 手写 SQL（见现有约定，禁用 `migrate dev` 的交互）。Docker 构建须 `NODE_ENV=production`；`DOCKER_BUILDKIT=0`（中文路径）。
- **生产构建门禁**：每个涉及 web 的任务结束后 `NODE_ENV=production next build` 必须 EXIT 0；跑完清 `web/.next` 并重启 web 恢复 dev。

```ts
import { prisma } from '../client'
import { decrypt } from '../crypto'

export type Capability = 'llm' | 'image' | 'tts' | 'asr'
export const CAPABILITIES: Capability[] = ['llm', 'image', 'tts', 'asr']

export type ResolvedCapConfig = {
  capability: Capability
  baseUrl: string
  apiKey: string
  model: string
  enabled: boolean
  extra: Record<string, unknown>
}

// 是否强制走 mock（未接通真实服务时用）
export function isMockMode(cfg: { enabled: boolean }): boolean {
  return process.env.AI_MOCK === '1' || !cfg.enabled
}

export async function getCapabilityConfig(cap: Capability): Promise<ResolvedCapConfig> {
  const row = await prisma.aiCapabilityConfig.findUnique({ where: { capability: cap } })
  return {
    capability: cap,
    baseUrl: row?.baseUrl ?? '',
    apiKey: row?.apiKeyEnc ? decrypt(row.apiKeyEnc) : '',
    model: row?.model ?? '',
    enabled: row?.enabled ?? false,
    extra: (row?.extra as Record<string, unknown> | undefined) ?? {},
  }
}
```

```ts
import { prisma, CAPABILITIES, encrypt } from '@mixcut/db'
import { requireRole, HttpError } from '@/lib/auth'
import { handler } from '@/lib/api'

export const PUT = handler(async (req, { params }) => {
  await requireRole('operator')
  const cap = params.capability
  if (!(CAPABILITIES as string[]).includes(cap)) throw new HttpError(404, '未知能力')
  const b = await req.json()
  if (b.enabled && !String(b.baseUrl ?? '').trim()) throw new HttpError(400, '开启前请先填写接口地址')
  const data: Record<string, unknown> = {
    baseUrl: String(b.baseUrl ?? '').trim(),
    model: String(b.model ?? '').trim(),
    enabled: !!b.enabled,
    extra: b.extra ?? undefined,
  }
  if (typeof b.apiKey === 'string' && b.apiKey.length > 0) data.apiKeyEnc = encrypt(b.apiKey)
  const row = await prisma.aiCapabilityConfig.upsert({
    where: { capability: cap },
    update: data,
    create: { capability: cap, ...data },
  })
  return NextResponse.json({ ok: true, hasKey: !!row.apiKeyEnc })
})
```

```ts
import { prisma } from './client'

export async function transitionRender(renderTaskId: string, to: string, note?: string): Promise<void> {
  const t = await prisma.renderTask.findUniqueOrThrow({ where: { id: renderTaskId } })
  await prisma.$transaction([
    prisma.renderTask.update({ where: { id: renderTaskId }, data: { status: to } }),
    prisma.renderStatusLog.create({ data: { renderTaskId, fromStatus: t.status, toStatus: to, note } }),
  ])
}

export async function setGenerationStatus(genTaskId: string, to: string): Promise<void> {
  await prisma.generationTask.update({ where: { id: genTaskId }, data: { status: to } })
}

export async function setSourceStatus(sourceVideoId: string, to: string): Promise<void> {
  await prisma.sourceVideo.update({ where: { id: sourceVideoId }, data: { status: to } })
}

export async function writeQc(renderTaskId: string, checkType: string, result: string, detail?: string): Promise<void> {
  await prisma.renderQcReport.create({ data: { renderTaskId, checkType, result, detail } })
}
```
