# v2.2 P2 拆解流水线 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** 跑通拆解流水线：运营粘抖音分享链接（或手动上传视频兜底）→ 下载 → ASR 转写 → 场景检测 → LLM 提炼可复用文案框架 → 存入框架库（FRAMEWORK_READY），供 P1 生成线使用。

**Architecture:** 沿用 monorepo。复用 P0 的 AI 适配层（`asrTranscribe`/`llmComplete`，mock 优先）、`renderState`（`setSourceStatus`）、`genQueue`（generation 队列 + gen worker）。P2 在同一 `generation` 队列/worker 增加 4 个拆解 job（payload 用 `sourceVideoId`），新增 `/api/extract/*` 路由、`/admin/extract` 与 `/admin/frameworks` 页。`/api/frameworks` GET 与 `/api/frameworks/[id]` GET/PATCH 已在 P1 存在，直接复用。仍**纯增量**。

**Tech Stack:** TypeScript、BullMQ、ffmpeg（音频抽取 + 场景检测 scene filter）、Next.js App Router、vitest。

**参考文档：** `docs/superpowers/specs/2026-07-16-video-deconstruct-generate-v2.2-design.md`（§4 SourceVideo/Transcript/SceneCut/CopyFramework、§5.1 拆解状态机、§6 jobs、§8 阈值）。

## Global Constraints

- **纯增量**：不改/不删旧混剪、P0、P1 生成线的模型/接口/页面/job。新 job/路由/页并存。`next build` 与 `vitest` 始终通过。
- **mock 优先**：拆解全流程在 mock（`asrTranscribe` 返回假转写、`llmComplete` 返回假框架文案）下跑通，产出一个真实 CopyFramework 行；场景检测用**真 ffmpeg**（对真实上传视频有效）。接真 ASR/LLM 靠模型配置页启用能力 + 填 key。
- **抖音入口**：`POST /api/extract/upload`（手动上传 mp4，**可靠主路径**）+ `POST /api/extract/from-link`（自研解析分享链，**best-effort**，失败则明确提示改用手动上传，不静默）。
- **拆解状态机**（source_videos.status，spec §5.1）：`CREATED → DOWNLOADING → TRANSCRIBING → SCENE_DETECTING → FRAMEWORK_EXTRACTING → FRAMEWORK_READY`；任一 job 失败 → `FAILED` + note。手动上传直接进 DOWNLOADING 完成态→TRANSCRIBING（跳过 download-douyin）。
- **产出框架默认**：`visualStyleType='ai_illustration'`、`renderTemplate='booklist'`、`overlayTemplate` 用书单号默认 `{"title_card":"{{标题}} {{副标题}}","watermark":"{{账号}}"}`，与 P1 渲染模板对齐。
- **worker 改动**后 `docker compose restart worker`；prisma 无新迁移（P0 已建表）。web 任务跑生产构建门禁（EXIT 0）后 `rm -rf web/.next && docker compose restart web` 恢复 dev。
- 全部 `/api/extract*` 路由 `requireRole('operator')`；上传文件路径用 uuid，不用用户文件名（防穿越）。

### Task 1: 拆解队列接入 + 手动上传入口 + 拆解详情接口

**Interfaces:**
- Produces: `enqueueGen` 支持 extract job（payload `{sourceVideoId}`）；`POST /api/extract/upload`（multipart mp4 → 建 SourceVideo(DOWNLOADING, videoFileUrl) → `enqueueGen('transcribe',{sourceVideoId})`）；`GET /api/extract/[id]`（source + transcript + sceneCut + frameworks）。

- [ ] **Step 1: genQueue 扩展**：
```ts
export type GenJobName =

  | 'align-captions' | 'render-visuals' | 'render-video' | 'run-gen-qc'
  | 'download-douyin' | 'transcribe' | 'detect-scenes' | 'extract-framework'
// payload 类型加 sourceVideoId?:
export async function enqueueGen(
  name: GenJobName,
  payload: { genTaskId?: string; renderTaskId?: string; sourceVideoId?: string },
): Promise<void> { /* 同现有 */ }
```
- [ ] **Step 2: gen worker 失败处理加 sourceVideoId 分支**（`worker/src/gen/index.ts`）：import `setSourceStatus`（已从 `@mixcut/db` 可导出）；在 failure handler 追加分支：
```ts
if (d.renderTaskId) await transitionRender(d.renderTaskId, 'FAILED', `${j?.name}: ${err.message}`).catch(() => {})
else if (d.sourceVideoId) await setSourceStatus(d.sourceVideoId, 'FAILED').catch(() => {})

```

- [ ] **Step 3: 上传入口** `web/app/api/extract/upload/route.ts`（仿 `web/app/api/materials/route.ts` 的 multipart）：
  - `requireRole('operator')`；`req.formData()` 取 file（校验 video/*）；存 `/data/source/<uuid>.mp4`；
  - `prisma.sourceVideo.create({ data:{ douyinShareUrl:'(manual-upload)', videoFileUrl:'/api/files/source/<uuid>.mp4', status:'DOWNLOADING', createdBy:session.userId }})`；
  - `enqueueGen('transcribe', { sourceVideoId })`（跳过 download-douyin）；返回 `{id}`。
- [ ] **Step 4: 拆解详情接口** `web/app/api/extract/[id]/route.ts`：`GET` operator-gated，返回 source + `transcripts`(最新) + `sceneCuts` + `frameworks`（该 source 产出的框架列表 id/name/status）。
- [ ] **Step 5: tsc + 冒烟**：上传一个 ffmpeg 造的 mp4（`ffmpeg -f lavfi -i testsrc=d=6:s=320x240 -f lavfi -i sine=d=6 -shortest x.mp4`），POST upload（operator cookie）→ 得 id → GET /api/extract/[id] 看到 source status（transcribe 未实现会 default throw→FAILED，正常）。验证 SourceVideo 行 + videoFileUrl。清理。
- [ ] **Step 6: Commit** `feat(extract): 拆解队列 payload + 手动上传入口 + 详情接口`。

### Task 2: download-douyin（分享链解析 best-effort + from-link 入口）

**Files:** `worker/src/gen/downloadDouyin.ts`、`web/app/api/extract/from-link/route.ts`、dispatch 接入

**Interfaces:**
- Consumes: `prisma.sourceVideo`(douyinShareUrl)、`setSourceStatus`、`enqueueGen`、`DATA_DIR`
- Produces: `POST /api/extract/from-link`（body `{shareUrl}` → 建 SourceVideo(CREATED, douyinShareUrl) → `enqueueGen('download-douyin',{sourceVideoId})` → `{id}`）；`downloadDouyin(sourceVideoId)`：best-effort 解析分享链→直链→下载到 `/data/source/<id>.mp4`→写 videoFileUrl + `setSourceStatus('TRANSCRIBING')` + `enqueueGen('transcribe')`；解析/下载失败 → `setSourceStatus('FAILED')` + note「链接解析失败，请改用手动上传」并抛错。

- [ ] **Step 1: from-link 路由**：operator-gated；校验 shareUrl 是字符串且含 http；建 SourceVideo(status CREATED)；enqueueGen('download-douyin')；返回 id。
- [ ] **Step 2: downloadDouyin.ts（best-effort，务实实现）**：
  - `setSourceStatus('DOWNLOADING')`；
  - 从 douyinShareUrl 里正则抽出短链（`https?://v.douyin.com/\w+` 等）；`fetch(shareUrl, { redirect:'follow' })` 跟随跳转拿到最终 URL，从中抽 `video_id`/`item_ids`；尝试构造无水印直链（已知 pattern，如 `https://aweme.snssdk.com/aweme/v1/play/?video_id=<id>&ratio=720p&line=0`）；`fetch` 直链 → 写 `/data/source/<id>.mp4`；校验文件是可用视频（ffprobe 有视频流，复用 Task 3 的 probe 或简单 ffprobe）。
  - 成功 → 写 videoFileUrl + `setSourceStatus('TRANSCRIBING')` + `enqueueGen('transcribe')`。
  - 任一步失败 → `setSourceStatus('FAILED')`（note 记录「解析失败，请手动上传」）+ throw。
  - **注**：抖音风控多变，此为 best-effort；可靠路径是手动上传。代码要健壮兜底、不 crash worker。
- [ ] **Step 3: dispatch 接入** `case 'download-douyin': return downloadDouyin(job.data.sourceVideoId)`。
- [ ] **Step 4: 冒烟**：from-link 传一个明显无效的链接 → 走到 FAILED + note（验证兜底不炸）。真实抖音链解析成功与否取决于风控，不作为验收硬指标（验收看手动上传主路径）。
- [ ] **Step 5: Commit**。

### Task 4: detect-scenes（ffmpeg 场景检测）

**Files:** `worker/src/ffmpeg.ts`（加 `detectScenes`）、`packages/db/src/scenes.ts`（纯解析 + 测试）、`worker/src/gen/detectScenes.ts`、dispatch 接入

**Interfaces:**
- Consumes: 视频、ffmpeg、`prisma.sceneCut`、`setSourceStatus`、`enqueueGen`
- Produces: `parseSceneCuts(ffmpegOutput): number[]`（ms，解析 showinfo 的 pts_time）；`detectScenes(video): Promise<string>`（跑 ffmpeg 抓输出）；job：cut_points_ms 写 `SceneCut` → `FRAMEWORK_EXTRACTING` + `enqueueGen('extract-framework')`。

- [ ] **Step 1: scenes.ts 纯解析 + 测试**：`parseSceneCuts(output: string): number[]` —— 从 `select='gt(scene,0.3)',showinfo` 的 stderr 里正则 `pts_time:([0-9.]+)` 抽出时间点（秒）→ ms 整数，去重升序。测试：给构造的 showinfo 输出 → 正确 ms 数组；无场景变化 → 空数组。
- [ ] **Step 2: detectScenes（ffmpeg.ts）**：`ffmpeg -i <video> -filter:v "select='gt(scene,0.3)',showinfo" -f null -`（抓 stderr）。
- [ ] **Step 3: detectScenes.ts job**：跑 detectScenes → parseSceneCuts → `prisma.sceneCut.create({ data:{ sourceVideoId, cutPointsMs }})`（可能为空数组，允许）→ `setSourceStatus('FRAMEWORK_EXTRACTING')` + `enqueueGen('extract-framework')`。
- [ ] **Step 4: dispatch 接入 + 测试 + 冒烟**：真实上传视频跑到 FRAMEWORK_EXTRACTING + SceneCut 行（cutPointsMs 数组，testsrc 视频通常有若干切点或空）。**Commit**。

### Task 5: extract-framework（LLM 提炼框架）

**Files:** `worker/src/gen/extractFramework.ts`、dispatch 接入

**Interfaces:**
- Consumes: `llmComplete`（P0，mock 返回假框架文案）、`prisma.transcript`/`sceneCut`、`prisma.copyFramework`、`setSourceStatus`
- Produces: `extractFramework(sourceVideoId)`：读 transcript + sceneCut → 组 prompt 让 LLM 产「框架结构描述 + 行业标签 + 建议分段数」→ 代码按 场景数/转写长度 估算 `suggestedSegmentCount`/`maxLines`/`maxTotalChars`（spec §8：从参考视频节奏反推）→ 建 `CopyFramework`（sourceVideoId、name、industryCategory、frameworkText、上述阈值、`imageStylePrompt` 默认一段风格词、`overlayTemplate` 书单号默认、`renderTemplate='booklist'`、`visualStyleType='ai_illustration'`、createdBy）→ `setSourceStatus('FRAMEWORK_READY')`。

- [ ] **Step 1: extractFramework.ts**：
  - 读最新 transcript.fullText、sceneCut.cutPointsMs；
  - `segCount = max(3, min(12, cutPointsMs.length || estimateFromSentences))`（场景切点数为主，无则按转写句数/默认 8）；
  - prompt：给 LLM 转写全文 + 段落节奏（段数），要求输出「可复用文案框架：开头句式/段落逻辑/语言风格/建议分段数」+ 行业标签（书单号/好物推荐/情感语录/知识科普）；`const frameworkText = await llmComplete({ system, prompt })`；行业标签可从 LLM 输出解析或简单默认 '书单号'；
  - 阈值估算（代码）：`maxLines = segCount + 2`；`maxTotalChars` 从 transcript 长度反推（如 `Math.min(600, max(120, Array.from(fullText).length))`，无转写则默认 220）；
  - `imageStylePrompt` 默认 '治愈系水彩插画，暖色调，柔和光线，统一画风'；
  - 建 CopyFramework（name 用 `拆解框架-<时间/subject 无则默认>`；overlayTemplate 书单号默认）；
  - `setSourceStatus('FRAMEWORK_READY')`。
- [ ] **Step 2: dispatch 接入**。
- [ ] **Step 3: 冒烟（端到端 mock）**：从 Task 1 上传 → transcribe → detect-scenes → extract-framework → SourceVideo=FRAMEWORK_READY + 一条 CopyFramework 行（frameworkText 非空、renderTemplate=booklist、阈值已填）。清理。**Commit**。

## P2 验收标准
1. 4 个拆解 job 全部接入 `generation` 队列，dispatch 无「未接入」抛错；失败正确置 SourceVideo FAILED。
2. 手动上传主路径：上传 mp4 → 端到端走到 FRAMEWORK_READY，产出一条可用 CopyFramework（frameworkText 非空、renderTemplate='booklist'、阈值已填）。from-link 为 best-effort，无效链优雅落 FAILED（不 crash）。
3. `/admin/extract`+`/[id]`+`/admin/frameworks` 可用；发起/进度/结果/框架编辑均通。
4. **拆解→生成闭环**：拆解产出的框架能被 P1 `/admin/generate` 用来出片（EXPORTED）。
5. `npm test` 全绿（P1 42 + scenes 新测试）；生产构建 EXIT 0。
6. 纯增量：旧混剪 + P0 + P1 生成线不受影响。
